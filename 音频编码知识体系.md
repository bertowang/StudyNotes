### 1.1 有损 vs 无损压缩

| 类型 | 原理 | 典型格式 | 压缩比 | 适用场景 |
|-----|------|---------|-------|---------|
| **无损压缩** | 统计冗余去除，可完美还原 | FLAC、ALAC、APE | 2:1 ~ 3:1 | 专业录音、母带存档 |
| **有损压缩** | 利用心理声学丢弃不可感知信息 | MP3、AAC、Opus | 10:1 ~ 20:1 | 流媒体、移动设备 |
| **语音编码** | 声道模型（LPC/CELP），专为人声优化 | G.729、AMR、EVS | 50:1 ~ 100:1 | 电话、VoIP |

## 二、必备数学基础

### 2.1 傅里叶变换（频域分析核心）

傅里叶变换是音频编码的数学基石——它告诉我们：**任何信号都可以分解为不同频率正弦波的叠加**。

#### 2.1.1 连续傅里叶变换

```
傅里叶变换（Continuous Fourier Transform）：

  正变换（时域 → 频域）：
  X(f) = ∫₋∞^∞ x(t) · e^(-j2πft) dt

  逆变换（频域 → 时域）：
  x(t) = ∫₋∞^∞ X(f) · e^(j2πft) df

  直觉理解：
  ├── X(f) 是复数：|X(f)| 是幅度谱，∠X(f) 是相位谱
  ├── |X(f)|² 是功率谱密度（PSD）
  └── 音频编码主要关注幅度谱（人耳对相位不敏感）

  帕塞瓦尔定理（能量守恒）：
  ∫|x(t)|² dt = ∫|X(f)|² df
  └── 时域总能量 = 频域总能量
      用于：验证编码前后能量损失
```

#### 2.1.2 离散傅里叶变换（DFT）

```
DFT（Discrete Fourier Transform）：

  N 点 DFT：
  X[k] = Σₙ₌₀^(N-1) x[n] · e^(-j2πkn/N)，k = 0,1,...,N-1

  逆 DFT（IDFT）：
  x[n] = (1/N) Σₖ₌₀^(N-1) X[k] · e^(j2πkn/N)

  复杂度：直接计算 O(N²)

FFT（快速傅里叶变换）：
  利用 DFT 的对称性和周期性，将复杂度降至 O(N log N)
  
  Cooley-Tukey 算法（最常用）：
  ├── 将 N 点 DFT 分解为两个 N/2 点 DFT（分治）
  ├── 蝶形运算：X[k] = E[k] + W_N^k · O[k]
  │   其中 W_N = e^(-j2π/N)（旋转因子）
  └── 递归分解直到 2 点 DFT

  实际应用：
  ├── 音频频谱分析（Audacity、Adobe Audition）
  ├── 均衡器（EQ）实现
  ├── 卷积混响（频域乘法 = 时域卷积）
  └── 心理声学模型的频率分析
```

#### 2.1.3 短时傅里叶变换（STFT）

```
STFT（Short-Time Fourier Transform）：

  问题：音频信号是非平稳的（频率随时间变化）
  解决：对信号加窗后做 DFT，得到时频表示

  STFT(τ, f) = ∫ x(t) · w(t-τ) · e^(-j2πft) dt

  其中 w(t) 是窗函数（Hann、Hamming、Blackman 等）

  参数选择：
  ├── 窗长（Frame Size）：
  │   ├── 长窗（2048 点）：频率分辨率高，时间分辨率低
  │   └── 短窗（256 点）：时间分辨率高，频率分辨率低
  └── 帧移（Hop Size）：相邻帧的重叠量（通常 50%~75%）

  时频不确定性原理（Heisenberg-Gabor 不等式）：
  Δt · Δf ≥ 1/(4π)
  └── 时间分辨率和频率分辨率不能同时任意高
      这是音频编码中"瞬态 vs 稳态"处理的根本矛盾

  声谱图（Spectrogram）：
  S(τ, f) = |STFT(τ, f)|²
  └── 可视化音频的时频能量分布
      是音频分析、机器学习特征提取的基础
```

### 2.2 离散余弦变换（DCT）

DCT 是 MP3、AAC 等编解码器的核心变换，比 DFT 更适合音频编码。

```
DCT-II（最常用，通常直接称为 DCT）：

  X[k] = Σₙ₌₀^(N-1) x[n] · cos(π(2n+1)k / (2N))，k = 0,1,...,N-1

  为什么 DCT 比 DFT 更适合音频编码？

  1. 实数输出：DCT 输出全为实数，无需存储虚部
     → 节省 50% 存储空间

  2. 能量集中性（Energy Compaction）：
     DCT 将信号能量集中在少数低频系数
     → 大多数高频系数接近零，可以量化为零（压缩）

  3. 无块效应（Block Effect）：
     DCT 的基函数在边界处自然满足对称条件
     → 比 DFT 的块效应更小

  MDCT（改进离散余弦变换）：
  X[k] = Σₙ₌₀^(2N-1) x[n] · cos(π/N · (n + N/2 + 1/2) · (k + 1/2))
  k = 0,1,...,N-1

  MDCT 的关键特性：
  ├── 50% 重叠：相邻帧重叠 50%，消除块边界伪影
  ├── 临界采样：N 个输入 → N/2 个输出（2:1 压缩）
  ├── 完美重建：IMDCT + 重叠相加（OLA）可完美还原
  └── 应用：MP3（混合滤波器组）、AAC（纯 MDCT）、Opus（CELT 模式）
```

### 2.3 小波变换

```
小波变换（Wavelet Transform）：

  核心思想：用可伸缩、可平移的"小波"函数分析信号
  
  连续小波变换（CWT）：
  W(a, b) = (1/√a) ∫ x(t) · ψ*((t-b)/a) dt
  
  其中：
  ├── ψ(t)：母小波（Morlet、Mexican Hat、Daubechies 等）
  ├── a：尺度参数（控制频率，a 大 → 低频）
  └── b：平移参数（控制时间位置）

  小波 vs STFT：
  ┌──────────────┬──────────────────┬──────────────────┐
  │              │      STFT        │      小波变换     │
  ├──────────────┼──────────────────┼──────────────────┤
  │ 窗口大小     │ 固定             │ 随频率自适应      │
  │ 低频分辨率   │ 时间分辨率低     │ 时间分辨率高      │
  │ 高频分辨率   │ 频率分辨率低     │ 频率分辨率高      │
  │ 适合信号     │ 平稳信号         │ 非平稳、瞬态信号  │
  └──────────────┴──────────────────┴──────────────────┘

  音频编码应用：
  ├── ATRAC（Sony MiniDisc）：使用 QMF 滤波器组（类小波）
  ├── DWT 用于音频水印嵌入
  └── 多分辨率分析：低频用长窗（高频率分辨率），高频用短窗（高时间分辨率）
      → AAC 的窗切换机制的理论基础
```

### 2.4 信息论基础

```
香农信息论（Shannon Information Theory）：

  信息熵（Entropy）：
  H(X) = -Σ p(x) · log₂ p(x)  [单位：比特/符号]
  
  直觉：熵 = 信号的平均不确定性 = 理论最小编码比特数
  
  示例：
  ├── 均匀分布（256 个等概率符号）：H = log₂(256) = 8 bit
  └── 偏斜分布（某符号概率 0.9）：H ≈ 0.47 bit（可大幅压缩）

  香农信道容量定理：
  C = B · log₂(1 + S/N)  [单位：bit/s]
  
  其中：
  ├── B：信道带宽（Hz）
  ├── S/N：信噪比（线性值）
  └── C：信道最大无误码传输速率

  率失真理论（Rate-Distortion Theory）：
  R(D) = min_{p(x̂|x): E[d(x,x̂)]≤D} I(X; X̂)
  
  含义：在允许失真 D 的条件下，最小编码速率 R(D)
  └── 音频有损编码的理论下界：给定比特率，最小可能失真是多少？

  Huffman 编码（最优前缀码）：
  ├── 高概率符号 → 短码字；低概率符号 → 长码字
  ├── 平均码长接近熵 H(X)
  └── 应用：MP3 的 Huffman 表（32 张预定义表）

  算术编码：
  ├── 将整个消息编码为 [0,1) 区间内的一个小数
  ├── 理论上可达到熵的极限（比 Huffman 更高效）
  └── 应用：AAC 的谱数据编码（部分模式）
```

### 2.5 滤波器理论

```
数字滤波器基础：

  FIR（有限冲激响应）滤波器：
  y[n] = Σₖ₌₀^(M) h[k] · x[n-k]
  
  特点：
  ├── 线性相位（不引入相位失真）
  ├── 无条件稳定
  └── 计算量大（需要大量乘加运算）

  IIR（无限冲激响应）滤波器：
  y[n] = Σₖ₌₀^M b[k]·x[n-k] - Σₖ₌₁^N a[k]·y[n-k]
  
  特点：
  ├── 计算效率高（阶数低）
  ├── 非线性相位（可能引入相位失真）
  └── 可能不稳定（需要检查极点位置）

  多相滤波器组（Polyphase Filter Bank）：
  └── 将宽带信号分解为多个子带
      MP3 使用 32 子带多相滤波器组
      计算效率：利用多相分解，复杂度降低 M 倍（M=子带数）

  QMF（正交镜像滤波器）：
  H₁(z) = H₀(-z)（高通 = 低通的镜像）
  
  完美重建条件：
  H₀(z)·H₀(z⁻¹) + H₀(-z)·H₀(-z⁻¹) = 2
  
  应用：
  ├── MPEG 音频子带分解
  ├── ATRAC（Sony）
  └── SBC（蓝牙基础音频编码）
```

### 2.6 线性预测编码（LPC）

```
LPC（Linear Predictive Coding）：

  核心思想：当前采样值可以由过去 p 个采样值线性预测
  
  预测模型：
  x̂[n] = Σₖ₌₁^p aₖ · x[n-k]
  
  预测误差（残差）：
  e[n] = x[n] - x̂[n]
  
  LPC 分析（求最优预测系数 aₖ）：
  最小化均方误差：E = Σ e²[n]
  
  → Yule-Walker 方程（Toeplitz 矩阵）：
  R · a = r
  其中 R[i,j] = R_xx[i-j]（自相关矩阵）
  
  Levinson-Durbin 递推算法：
  ├── 高效求解 Yule-Walker 方程
  ├── 复杂度：O(p²)（比直接求逆 O(p³) 快）
  └── 同时给出反射系数（PARCOR 系数）

  声道模型（Source-Filter Model）：
  x(z) = G · E(z) / A(z)
  
  其中：
  ├── E(z)：激励信号（声门脉冲/噪声）
  ├── A(z) = 1 - Σaₖz⁻ᵏ：声道全极点滤波器
  └── G：增益
  
  应用：
  ├── 语音编码（G.711、G.729、AMR）
  ├── CELP（码激励线性预测）：G.729、AMR-NB
  └── Opus 的 SILK 模式（语音层）

CELP（Code Excited Linear Prediction）：
  
  编码过程：
  1. LPC 分析 → 声道滤波器系数
  2. 长时预测（LTP）→ 基音周期（音调）
  3. 码本搜索：从固定码本中找最佳激励向量
  4. 量化并传输：LPC 系数 + 基音 + 码本索引 + 增益
  
  解码过程：
  码本激励 → 基音滤波器 → 声道滤波器 → 语音
  
  应用：G.729（8kbps）、AMR（4.75~12.2kbps）、Opus SILK 模式
```

## 三、心理声学模型

> 心理声学（Psychoacoustics）是音频有损压缩的理论基础——利用人耳的感知局限，丢弃"听不见"的信息。

### 3.1 人耳听觉特性

```
人耳基本参数：

  可听频率范围：20 Hz ~ 20,000 Hz（随年龄退化）
  可听动态范围：0 dB SPL（听阈）~ 120 dB SPL（痛阈）
  
  频率分辨率：
  ├── 低频（<500 Hz）：约 3 Hz 的频率差异可分辨
  └── 高频（>1 kHz）：约 0.3% 的相对频率差异可分辨
  
  时间分辨率：
  ├── 时间掩蔽前向：约 200ms（声音结束后仍有掩蔽）
  └── 时间掩蔽后向：约 5~20ms（声音开始前的掩蔽）
  
  双耳效应（Binaural Hearing）：
  ├── ITD（耳间时间差）：低频定位，精度约 10μs
  └── ILD（耳间强度差）：高频定位
```

### 3.2 掩蔽效应

```
掩蔽效应（Masking Effect）：

  同时掩蔽（Simultaneous Masking）：
  强音（掩蔽音）使同时存在的弱音（被掩蔽音）听不见
  
  掩蔽阈值（Masking Threshold）：
  ├── 掩蔽音越强 → 掩蔽阈值越高（更多信号被掩蔽）
  ├── 频率越接近 → 掩蔽效果越强
  └── 上行掩蔽 > 下行掩蔽（高频比低频更容易被掩蔽）
  
  时间掩蔽（Temporal Masking）：
  
  前向掩蔽（Post-masking）：
  ├── 强音结束后，掩蔽效果持续约 100~200ms
  └── 原因：听觉神经的恢复时间
  
  后向掩蔽（Pre-masking）：
  ├── 强音开始前约 5~20ms 内的弱音也被掩蔽
  └── 原因：大脑处理延迟（逆时间因果性！）
  
  掩蔽效应在编码中的应用：
  ├── 计算每个频带的掩蔽阈值
  ├── 量化噪声 < 掩蔽阈值 → 量化噪声听不见
  └── 比特分配：掩蔽阈值高的频带可以用更少比特
  
  信噪掩蔽比（SMR）：
  SMR[b] = 信号功率[b] / 掩蔽阈值[b]
  └── SMR 越高 → 该频带越需要更多比特
```

### 3.3 临界频带与 Bark 尺度

```
临界频带（Critical Band）：

  人耳将频率划分为约 24 个临界频带（Bark 频带）
  每个频带内的声音相互掩蔽最强
  
  Bark 尺度转换：
  z(f) = 13·arctan(0.00076f) + 3.5·arctan((f/7500)²)
  
  其中 f 单位为 Hz，z 单位为 Bark（0~24 Bark）
  
  临界频带宽度（ERB，等效矩形带宽）：
  ERB(f) = 24.7 · (4.37f/1000 + 1)  [Hz]
  
  低频（100 Hz）：ERB ≈ 35 Hz（窄带宽）
  高频（10 kHz）：ERB ≈ 1100 Hz（宽带宽）
  
  Mel 尺度（另一种感知频率尺度）：
  mel(f) = 2595 · log₁₀(1 + f/700)
  
  应用：
  ├── MFCC（梅尔频率倒谱系数）：语音识别特征
  ├── 音频神经网络的输入特征
  └── 感知音频质量评估（PEAQ）

  MP3/AAC 的频带划分：
  ├── MP3：32 个等宽子带（多相滤波器组）
  │   + 576 个 MDCT 系数（混合滤波器组）
  └── AAC：1024 个 MDCT 系数，按 Bark 尺度分组量化
```

### 3.4 等响曲线

```
等响曲线（Equal-Loudness Contours，Fletcher-Munson 曲线）：

  描述：人耳对不同频率的灵敏度不同
  
  关键特点：
  ├── 1 kHz 附近最灵敏（听阈最低）
  ├── 3~4 kHz 有共振峰（外耳道共振）
  ├── 低频（<200 Hz）需要更高声压才能听见
  └── 高频（>10 kHz）灵敏度急剧下降
  
  响度单位：
  ├── 声压级（SPL）：dB，物理量
  ├── 响度级（Loudness Level）：phon，感知量
  └── 响度（Loudness）：sone，线性感知量（1 sone = 40 phon）
  
  A 加权（A-weighting）：
  └── 模拟人耳频率响应的滤波器
      dBA = 声压级 + A 加权修正
      用于：噪声测量、音频质量评估

  编码应用：
  ├── 低频和高频可以用更粗糙的量化（人耳不敏感）
  ├── 3~4 kHz 需要最精细的量化
  └── 感知噪声整形（Noise Shaping）：将量化噪声推向不敏感频段
```

## 四、音频编码核心技术

### 4.1 量化与编码

```
均匀量化（Uniform Quantization）：

  量化步长：Δ = (x_max - x_min) / 2^B
  量化误差：e = x - Q(x)，|e| ≤ Δ/2
  量化噪声功率：σ²_q = Δ²/12
  
  信噪比（SQNR）：
  SQNR ≈ 6.02B + 1.76 dB（正弦信号）
  └── 每增加 1 bit，SQNR 提升约 6 dB
      CD 音质（16 bit）：SQNR ≈ 98 dB

非均匀量化（Non-uniform Quantization）：
  
  μ 律（μ-law，北美/日本）：
  F(x) = sgn(x) · ln(1 + μ|x|) / ln(1 + μ)，μ = 255
  
  A 律（A-law，欧洲）：
  F(x) = sgn(x) · A|x|/(1+lnA)，|x| ≤ 1/A
         sgn(x) · (1+ln(A|x|))/(1+lnA)，|x| > 1/A，A = 87.6
  
  优点：小信号量化精度高，大信号量化精度低
  应用：G.711（电话语音，8kHz 采样，64kbps）

感知量化（Perceptual Quantization）：
  
  核心：根据心理声学掩蔽阈值动态分配量化比特
  
  步骤：
  1. 计算各频带的信号功率
  2. 计算各频带的掩蔽阈值（心理声学模型）
  3. 比特分配：SMR 高的频带分配更多比特
  4. 量化：使量化噪声低于掩蔽阈值
  
  目标：量化噪声在感知上不可见（听不见）
```

### 4.2 子带编码

```
子带编码（Sub-Band Coding, SBC）：

  原理：
  1. 用滤波器组将信号分解为 M 个子带
  2. 对每个子带降采样（M 倍）
  3. 根据各子带的感知重要性分配比特
  4. 独立量化各子带
  5. 解码时重建各子带，合并输出
  
  优点：
  ├── 可针对每个子带独立控制量化精度
  ├── 利用心理声学掩蔽效应
  └── 计算效率高（多相滤波器组）
  
  缺点：
  └── 子带间的频率分辨率固定（不如 MDCT 灵活）
  
  应用：
  ├── MP3 的第一级分解（32 子带多相滤波器组）
  ├── SBC（蓝牙 A2DP 基础编码）
  └── MPEG-1 Layer I/II
```

### 4.3 变换编码

```
变换编码（Transform Coding）：

  原理：
  1. 将时域信号块变换到频域（DCT/MDCT）
  2. 频域系数具有能量集中性（大多数系数接近零）
  3. 根据感知重要性量化各频域系数
  4. 熵编码压缩量化后的系数
  
  MDCT 窗函数：
  
  长窗（Long Block，2048 点）：
  ├── 频率分辨率高（1024 个频率线）
  ├── 适合稳态信号（乐器持续音）
  └── 时间分辨率低（可能产生预回声）
  
  短窗（Short Block，256 点）：
  ├── 时间分辨率高（128 个频率线）
  ├── 适合瞬态信号（打击乐、爆破音）
  └── 频率分辨率低
  
  窗切换（Window Switching）：
  ├── 检测到瞬态 → 切换到短窗（避免预回声）
  └── 稳态信号 → 使用长窗（更好的频率分辨率）
  
  预回声（Pre-echo）问题：
  ├── 原因：长窗内的瞬态信号量化噪声扩散到整个窗
  ├── 表现：在瞬态之前听到"嗡嗡"声
  └── 解决：瞬态检测 + 窗切换 + 时间噪声整形（TNS）
```

### 4.4 预测编码

```
时域噪声整形（TNS，Temporal Noise Shaping）：

  问题：MDCT 变换后，量化噪声在时域均匀分布
        但信号能量集中在瞬态处 → 预回声
  
  解决：在频域对 MDCT 系数做线性预测
        使量化噪声的时域分布跟随信号包络
  
  原理：
  1. 对 MDCT 系数做 LPC 分析
  2. 用 LPC 滤波器对 MDCT 系数做预测
  3. 传输预测残差（而非原始系数）
  4. 解码时用 LPC 合成滤波器恢复
  
  效果：量化噪声集中在信号能量大的地方（被掩蔽）
  应用：AAC 标准的核心工具之一

立体声编码技术：

  独立立体声（Independent Stereo）：
  └── 左右声道独立编码，无压缩增益

  中/侧立体声（M/S Stereo）：
  ├── M = (L + R) / 2（中间声道）
  ├── S = (L - R) / 2（侧声道）
  └── 优点：S 声道能量通常小，可用更少比特

  强度立体声（Intensity Stereo）：
  ├── 高频段：只保留能量包络，丢弃相位差
  └── 原理：高频定位主要依赖 ILD，不依赖 ITD

  双耳提示编码（Binaural Cue Coding, BCC）：
  └── 参数化立体声，用少量参数描述空间感
      AAC+ 的 PS（参数立体声）基于此原理
```

### 4.5 熵编码

```
Huffman 编码（MP3 使用）：

  MP3 的 Huffman 表：
  ├── 32 张预定义 Huffman 表（ISO 13818-3 标准）
  ├── 编码器选择最优表（最短码字）
  └── 大值区（bigvalues）+ 小值区（count1）+ 零值区

  游程编码（Run-Length Encoding）：
  └── 连续零系数用（零的个数, 非零值）对表示
      MDCT 高频系数大多为零 → 高效压缩

算术编码（AAC 部分模式使用）：
  ├── 理论上可达到熵的极限
  └── 计算复杂度高于 Huffman

RANS（非对称数字系统，Opus 使用）：
  ├── 结合算术编码的效率和 Huffman 的速度
  └── Opus 1.1+ 使用 CELT 的范围编码（Range Coding）
```

## 五、MP3 编码技术详解

```
MP3（MPEG-1 Audio Layer III）
标准：ISO/IEC 11172-3（1993年）
```

### 5.1 MP3 编码流程

```
MP3 编码完整流程：

  输入 PCM（44.1kHz, 16bit, 立体声）
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  1. 多相滤波器组（Polyphase Filter Bank）            │
  │     32 个等宽子带，每子带 18 个 MDCT 系数            │
  │     总计：32 × 18 = 576 个频率线/声道               │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  2. MDCT（改进离散余弦变换）                         │
  │     长窗：每子带 18 个系数（频率分辨率高）            │
  │     短窗：每子带 6 个系数 × 3（时间分辨率高）         │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  3. 心理声学模型（Model I 或 Model II）              │
  │     计算每个频带的掩蔽阈值（SMR）                    │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  4. 量化与比特分配（双嵌套迭代循环）                  │
  │     外循环：调整全局增益，控制总比特数                │
  │     内循环：调整比例因子，使量化噪声 < 掩蔽阈值       │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  5. Huffman 编码                                     │
  │     从 32 张表中选最优表                             │
  │     大值区 + 小值区 + 零值区分别编码                 │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  6. 比特流打包（Frame 格式）                         │
  │     帧头（32bit）+ 侧信息 + 主数据 + 辅助数据        │
  └─────────────────────────────────────────────────────┘
       ↓
  输出 MP3 比特流
```

### 5.2 MP3 帧结构

```
MP3 帧格式（每帧 = 1152 个 PCM 采样）：

  ┌──────────┬──────────┬──────────────┬──────────┐
  │ 同步字   │  帧头    │   侧信息     │  主数据  │
  │ 11 bits  │ 21 bits  │ 17/32 bytes  │ 可变长度 │
  └──────────┴──────────┴──────────────┴──────────┘

  帧头字段：
  ├── MPEG 版本（MPEG-1/2/2.5）
  ├── Layer（I/II/III）
  ├── 比特率索引（32~320 kbps）
  ├── 采样率索引（32/44.1/48 kHz）
  ├── 声道模式（立体声/联合立体声/双声道/单声道）
  └── 版权/原创标志

  比特率 vs 质量（44.1kHz 立体声）：
  ├── 320 kbps：接近 CD 质量，大多数人无法区分
  ├── 192 kbps：良好质量，大多数场景足够
  ├── 128 kbps：可接受质量，流媒体常用
  └── 64 kbps：明显失真，仅适合语音

  MP3 的局限性：
  ├── 混合滤波器组（多相 + MDCT）引入额外复杂度
  ├── 固定 32 子带（不如纯 MDCT 灵活）
  ├── 无内置 SBR（频带复制）→ 低比特率高频损失严重
  └── 专利已于 2017 年到期（现已完全开放）
```

## 六、AAC 编码技术详解

```
AAC（Advanced Audio Coding）
标准：ISO/IEC 13818-7（1997年，MPEG-2 AAC）
      ISO/IEC 14496-3（1999年，MPEG-4 AAC）
设计目标：在相同质量下比 MP3 节省 30% 比特率
```

### 6.1 AAC 编码流程

```
AAC 编码完整流程：

  输入 PCM
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  1. 纯 MDCT 变换（无多相滤波器组）                   │
  │     长窗：2048 点 MDCT → 1024 个频率线              │
  │     短窗：256 点 MDCT × 8 → 128 × 8 个频率线        │
  │     窗切换：瞬态检测 → 自动切换长/短窗               │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  2. 时域噪声整形（TNS）                              │
  │     频域 LPC 预测，消除预回声                        │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  3. 强度/M-S 立体声编码                              │
  │     按频带选择最优立体声编码方式                     │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  4. 预测编码（AAC-LC 可选）                          │
  │     帧间预测，减少稳态信号的冗余                     │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  5. 感知量化（心理声学模型 II）                       │
  │     比特分配：基于 SMR 的迭代优化                    │
  └─────────────────────────────────────────────────────┘
       ↓
  ┌─────────────────────────────────────────────────────┐
  │  6. 无噪声编码（Noiseless Coding）                   │
  │     Huffman 编码 + 比例因子编码                      │
  └─────────────────────────────────────────────────────┘
       ↓
  输出 AAC 比特流（ADTS/ADIF/MP4 容器）
```

### 6.2 AAC 家族

```
AAC 编码器家族：

  AAC-LC（Low Complexity）：
  ├── 最基础的 AAC 配置
  ├── 适合比特率：64~320 kbps
  └── 应用：iTunes、YouTube、Apple Music

  HE-AAC v1（High Efficiency AAC，AAC+ 或 aacPlus）：
  ├── AAC-LC + SBR（频谱带宽复制）
  ├── SBR 原理：
  │   ├── 编码器只编码低频部分（节省比特）
  │   ├── 传输少量参数描述高频包络
  │   └── 解码器用低频信号"复制"高频（谐波扩展）
  ├── 适合比特率：24~64 kbps
  └── 应用：DAB+ 数字广播、低比特率流媒体

  HE-AAC v2（AAC++ 或 eAAC+）：
  ├── AAC-LC + SBR + PS（参数立体声）
  ├── PS 原理：
  │   ├── 只传输单声道信号
  │   └── 用少量参数（ILD、IPD、ICC）重建立体声感
  ├── 适合比特率：12~32 kbps
  └── 应用：极低比特率语音/音乐流媒体

  AAC-LD（Low Delay）：
  ├── 算法延迟 < 20ms（适合实时通信）
  └── 应用：FaceTime、Apple 实时音频

  AAC-ELD（Enhanced Low Delay）：
  ├── 算法延迟 < 15ms
  ├── 结合 SBR 的低延迟版本
  └── 应用：高质量实时通信

  xHE-AAC（Extended HE-AAC，MPEG-D USAC）：
  ├── 统一语音和音频编码（USAC）
  ├── 无缝处理语音和音乐混合内容
  ├── 适合比特率：12~64 kbps
  └── 应用：下一代流媒体、DAB+ 升级
```

## 七、Opus 编码技术详解

```
Opus
标准：RFC 6716（2012年，IETF）
设计目标：统一覆盖语音到音乐、低延迟到高质量的全场景编解码器
开源：完全开源，无专利费用
```

### 7.1 Opus 架构设计

```
Opus 的混合架构：

  Opus = SILK（语音层）+ CELT（音频层）

  ┌─────────────────────────────────────────────────────┐
  │                    Opus 编码器                       │
  │                                                     │
  │  ┌──────────────┐    ┌──────────────────────────┐  │
  │  │  SILK 模式   │    │      CELT 模式            │  │
  │  │（语音专用）   │    │   （宽带音频/低延迟）      │  │
  │  │              │    │                          │  │
  │  │ LPC/CELP 预测│    │ MDCT 变换编码             │  │
  │  │ 基音预测     │    │ 范围编码（Range Coding）  │  │
  │  │ 8~24 kHz    │    │ 8~48 kHz                 │  │
  │  └──────────────┘    └──────────────────────────┘  │
  │           ↘                    ↙                   │
  │         ┌──────────────────────┐                   │
  │         │    混合模式（Hybrid）  │                   │
  │         │  SILK 低频 + CELT 高频│                   │
  │         └──────────────────────┘                   │
  └─────────────────────────────────────────────────────┘

  模式自动切换：
  ├── 纯语音（<8 kbps）：SILK 模式
  ├── 语音/音乐混合（8~20 kbps）：混合模式
  └── 音乐/低延迟（>20 kbps）：CELT 模式
```

### 7.2 CELT 技术细节

```
CELT（Constrained Energy Lapped Transform）：

  核心算法：
  1. MDCT 变换（帧长 2.5~60ms 可变）
  2. 频带能量归一化（Band Normalization）
  3. 金字塔向量量化（PVQ，Pyramid Vector Quantization）
  4. 范围编码（Range Coding，算术编码变体）

  PVQ（金字塔向量量化）：
  ├── 将频带内的 MDCT 系数视为单位球面上的向量
  ├── 在 L1 范数球面（金字塔）上量化
  └── 优点：自然保持能量归一化，无需单独量化幅度

  范围编码（Range Coding）：
  ├── 算术编码的变体，效率接近熵极限
  ├── 比 Huffman 编码效率高约 0.5~1%
  └── 实现简单，适合硬件加速

  帧长灵活性：
  ├── 2.5ms：超低延迟（游戏语音）
  ├── 5ms：低延迟
  ├── 10ms：标准延迟（WebRTC 默认）
  ├── 20ms：高质量（流媒体）
  └── 60ms：最高压缩效率

  算法延迟：
  ├── CELT 模式：5ms（最低）
  └── SILK 模式：约 25ms
```

### 7.3 Opus 的带宽模式

```
Opus 带宽配置：

  NB（Narrowband）：8 kHz 采样，4 kHz 带宽
  └── 电话语音质量

  MB（Mediumband）：12 kHz 采样，6 kHz 带宽
  └── 增强语音质量

  WB（Wideband）：16 kHz 采样，8 kHz 带宽
  └── 高清语音（HD Voice）

  SWB（Super-Wideband）：24 kHz 采样，12 kHz 带宽
  └── 超宽带语音

  FB（Fullband）：48 kHz 采样，20 kHz 带宽
  └── 完整音乐质量

  比特率范围：6 kbps ~ 510 kbps
  
  典型配置：
  ├── 6 kbps：语音，可懂但质量差
  ├── 12 kbps：语音，良好质量
  ├── 24 kbps：语音，接近透明
  ├── 64 kbps：音乐，良好质量
  ├── 128 kbps：音乐，接近透明
  └── 256+ kbps：音乐，透明质量
```

### 8.3 延迟对比

```

  └── 约 60~80ms

  └── 约 20ms

  └── 约 15ms

  Opus（CELT 模式，10ms 帧）：
  └── 约 22.5ms（含 2.5ms 前瞻）

  Opus（CELT 模式，2.5ms 帧）：
  └── 约 5ms
      适合超低延迟游戏语音
```

```
  # 对比两个音频文件的差异
  ffmpeg -i original.wav -i encoded.mp3 -filter_complex "[0][1]amix=inputs=2:duration=first:dropout_transition=0,aeval='-val(0)':c=same[diff]" -map "[diff]" diff.wav

libopus（Opus 官方库）：
  # 安装
  sudo apt install libopus-dev
```

```python
def analyze_spectrum(audio_file, title="频谱分析"):
    """分析音频文件的频谱特性"""
    y, sr = librosa.load(audio_file, sr=None, mono=True)
    
    fig, axes = plt.subplots(3, 1, figsize=(12, 10))
    fig.suptitle(title, fontsize=14)

    librosa.display.waveshow(y, sr=sr, ax=axes[0])
    axes[0].set_title("波形（时域）")
    axes[0].set_xlabel("时间 (s)")
    axes[0].set_ylabel("幅度")
    
    # 2. 频谱图（STFT）
    D = librosa.stft(y, n_fft=2048, hop_length=512)
    S_db = librosa.amplitude_to_db(np.abs(D), ref=np.max)
    img = librosa.display.specshow(S_db, sr=sr, hop_length=512,
                                    x_axis='time', y_axis='hz', ax=axes[1])
    axes[1].set_title("频谱图（STFT，dB）")
    plt.colorbar(img, ax=axes[1], format="%+2.0f dB")
    
    # 3. 梅尔频谱图（感知频率尺度）
    M = librosa.feature.melspectrogram(y=y, sr=sr, n_mels=128)
    M_db = librosa.power_to_db(M, ref=np.max)
    img2 = librosa.display.specshow(M_db, sr=sr, x_axis='time',
                                     y_axis='mel', ax=axes[2])
    axes[2].set_title("梅尔频谱图（感知频率尺度）")
    plt.colorbar(img2, ax=axes[2], format="%+2.0f dB")

    plt.show()
    
    print(f"采样率: {sr} Hz")
    print(f"时长: {len(y)/sr:.2f} 秒")
    print(f"动态范围: {20*np.log10(np.max(np.abs(y))/np.mean(np.abs(y))):.1f} dB")
    return y, sr

def compare_codecs(input_wav, bitrate=128):
    """对比 MP3、AAC、Opus 在相同比特率下的质量"""
    
    # 编码为不同格式
    formats = {
        'MP3':  (f'output_{bitrate}k.mp3',  f'libmp3lame -b:a {bitrate}k'),
        'AAC':  (f'output_{bitrate}k.m4a',  f'aac -b:a {bitrate}k'),
        'Opus': (f'output_{bitrate}k.opus', f'libopus -b:a {bitrate}k'),
    }
    
    results = {}
    for name, (output_file, codec_args) in formats.items():
        # 编码
        cmd = f'ffmpeg -y -i {input_wav} -codec:a {codec_args} {output_file}'
        subprocess.run(cmd, shell=True, capture_output=True)
        
        # 解码回 WAV
        decoded_wav = f'decoded_{name.lower()}_{bitrate}k.wav'
        cmd2 = f'ffmpeg -y -i {output_file} {decoded_wav}'
        subprocess.run(cmd2, shell=True, capture_output=True)
        
        # 计算文件大小
        size_kb = os.path.getsize(output_file) / 1024
        
        # 计算 SNR
        orig, sr = librosa.load(input_wav, sr=None, mono=True)
        dec, _   = librosa.load(decoded_wav, sr=sr, mono=True)
        
        # 对齐长度
        min_len = min(len(orig), len(dec))
        orig, dec = orig[:min_len], dec[:min_len]
        
        noise = orig - dec
        snr = 10 * np.log10(np.sum(orig**2) / (np.sum(noise**2) + 1e-10))
        
        results[name] = {
            'file_size_kb': size_kb,
            'snr_db': snr,
            'decoded_wav': decoded_wav
        }
        print(f"{name:6s}: 文件大小={size_kb:.1f}KB, SNR={snr:.1f}dB")
    
    return results

# 3. 心理声学掩蔽阈值可视化

def plot_masking_threshold(freq_hz=1000, level_db=60):
    """可视化心理声学掩蔽阈值（简化模型）"""
    freqs = np.logspace(np.log10(20), np.log10(20000), 1000)
    
    # 绝对听阈（ISO 226）
    f = freqs / 1000  # kHz
    abs_threshold = (3.64 * f**(-0.8) 
                     - 6.5 * np.exp(-0.6*(f-3.3)**2) 
                     + 1e-3 * f**4)
    
    # 简化掩蔽阈值（以 freq_hz 为掩蔽音）
    bark_masker = 13 * np.arctan(0.00076 * freq_hz) + 3.5 * np.arctan((freq_hz/7500)**2)
    bark_freqs  = 13 * np.arctan(0.00076 * freqs)   + 3.5 * np.arctan((freqs/7500)**2)
    
    delta_bark = bark_freqs - bark_masker
    
    # 掩蔽扩展函数（简化）
    masking_spread = np.where(
        delta_bark >= 0,
        level_db - 27 * delta_bark,           # 上行掩蔽（较弱）
        level_db + 25 * delta_bark            # 下行掩蔽（较强）
    )
    
    masking_threshold = np.maximum(abs_threshold, masking_spread)
    
    plt.figure(figsize=(12, 6))
    plt.semilogx(freqs, abs_threshold, 'b--', label='绝对听阈', linewidth=1.5)
    plt.semilogx(freqs, masking_spread, 'g-', label=f'掩蔽扩展 ({freq_hz}Hz, {level_db}dB)', linewidth=1.5)
    plt.semilogx(freqs, masking_threshold, 'r-', label='综合掩蔽阈值', linewidth=2)
    plt.axvline(x=freq_hz, color='orange', linestyle=':', label=f'掩蔽音 {freq_hz}Hz')
    
    plt.xlabel('频率 (Hz)')
    plt.ylabel('声压级 (dB SPL)')
    plt.title('心理声学掩蔽阈值（量化噪声低于此线则听不见）')
    plt.legend()
    plt.grid(True, alpha=0.3)
    plt.xlim([20, 20000])
    plt.ylim([-20, 100])
    plt.tight_layout()
    plt.savefig('masking_threshold.png', dpi=150)
    plt.show()

# 4. MDCT 演示
# ============================================================
def demo_mdct(signal_length=1024):
    """演示 MDCT 的能量集中性"""
    # 生成测试信号（440Hz 正弦波 + 噪声）
    t = np.linspace(0, 1, signal_length)
    signal = np.sin(2 * np.pi * 440 * t) + 0.1 * np.random.randn(signal_length)
    
    # DCT-II（近似 MDCT）
    from scipy.fft import dct
    dct_coeffs = dct(signal, type=2, norm='ortho')
    
    # 能量集中性分析
    total_energy = np.sum(dct_coeffs**2)
    sorted_coeffs = np.sort(np.abs(dct_coeffs))[::-1]
    cumulative_energy = np.cumsum(sorted_coeffs**2) / total_energy
    
    # 找到包含 99% 能量所需的系数数量
    n_99 = np.argmax(cumulative_energy >= 0.99) + 1
    print(f"包含 99% 能量只需 {n_99}/{signal_length} 个系数 "
          f"（压缩比 {signal_length/n_99:.1f}:1）")
    
    fig, axes = plt.subplots(2, 2, figsize=(12, 8))
    
    axes[0,0].plot(t, signal)
    axes[0,0].set_title("原始信号（时域）")
    axes[0,0].set_xlabel("时间 (s)")
    
    axes[0,1].stem(np.abs(dct_coeffs[:200]), markerfmt='C0.', linefmt='C0-', basefmt='k-')
    axes[0,1].set_title("DCT 系数（前200个）")
    axes[0,1].set_xlabel("系数索引")
    
    axes[1,0].plot(cumulative_energy)
    axes[1,0].axhline(y=0.99, color='r', linestyle='--', label='99% 能量')
    axes[1,0].axvline(x=n_99, color='g', linestyle='--', label=f'{n_99} 个系数')
    axes[1,0].set_title("累积能量分布（能量集中性）")
    axes[1,0].set_xlabel("系数数量")
    axes[1,0].set_ylabel("累积能量比例")
    axes[1,0].legend()
    
    # 量化后重建（保留前 n_99 个最大系数）
    threshold = sorted_coeffs[n_99-1]
    compressed_coeffs = np.where(np.abs(dct_coeffs) >= threshold, dct_coeffs, 0)
    from scipy.fft import idct
    reconstructed = idct(compressed_coeffs, type=2, norm='ortho')
    
    axes[1,1].plot(t, signal, label='原始', alpha=0.7)
    axes[1,1].plot(t, reconstructed, label=f'重建（{n_99}个系数）', alpha=0.7)
    axes[1,1].set_title("原始 vs 重建信号")
    axes[1,1].set_xlabel("时间 (s)")
    axes[1,1].legend()
    
    plt.tight_layout()
    plt.savefig('mdct_demo.png', dpi=150)
    plt.show()
```

### 10.1 神经音频编解码器

```
传统编解码器 vs 神经编解码器：

  传统：手工设计的心理声学模型 + 变换编码
  神经：端到端神经网络学习最优编码表示

主要神经音频编解码器：

  Encodec（Meta，2022）：
  ├── 架构：卷积编码器 + 残差向量量化（RVQ）+ 卷积解码器
  ├── 比特率：1.5 ~ 24 kbps
  ├── 特点：极低比特率下质量优于传统编解码器
  └── 开源：https://github.com/facebookresearch/encodec

  SoundStream（Google，2021）：
  ├── 架构：类似 Encodec，流式处理
  ├── 比特率：3 ~ 18 kbps
  └── 应用：Google Meet 音频增强

  DAC（Descript Audio Codec，2023）：
  ├── 改进的 RVQ，更高质量
  └── 开源：https://github.com/descriptinc/descript-audio-codec

  AudioDec（2023）：
  └── 实时神经音频编解码器，低延迟

残差向量量化（RVQ）：
  1. 第一个码本量化输入向量 → 残差1
  2. 第二个码本量化残差1 → 残差2
  3. 依此类推，N 个码本级联
  
  优点：
  ├── 比特率可调（使用更少码本 = 更低比特率）
  └── 质量随码本数量平滑提升
```

> **总结**：音频编码是数学、信号处理、心理声学和工程实现的完美结合。核心是**利用人耳的感知局限（掩蔽效应）丢弃"听不见"的信息**。MP3 奠定了感知编码的基础，AAC 在相同质量下节省了 30% 比特率，Opus 则统一了语音和音乐编码，并以开源免费的方式成为互联网音频的新标准。AI 时代，神经编解码器（Encodec/SoundStream）正在挑战传统方法，在极低比特率下实现了前所未有的音质。
