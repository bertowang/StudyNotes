## 0. 写在最前：为什么训练大模型必须懂这三剑客

### 0.1 单卡训 7B 的窘境

用 PyTorch DDP 训 7B 模型（fp16 + AdamW）需要的显存：

| 项 | 大小 |
|:--|:--|
| 模型参数（fp16）| 14 GB |
| 梯度（fp16）| 14 GB |
| 优化器状态（fp32 + m + v）| **84 GB** |
| Activation（seq=2K, bsz=1）| ~20 GB |
| **合计** | **~132 GB** |

**问题**：H100 只有 80 GB，A100 只有 40/80 GB —— **单卡根本训不动 7B**。

### 0.2 三剑客解决什么问题

| 框架 | 核心技术 | 一句话定位 |
|:--|:--|:--|
| **DeepSpeed** | ZeRO 1/2/3 | 微软出品，**切参数/梯度/优化器状态**，最省显存 |
| **FSDP** | Fully Sharded DP | PyTorch 原生 ZeRO-3，**API 更清爽** |
| **Megatron-LM** | TP + PP + DP（3D 并行）| NVIDIA 出品，**切模型本身**，极致性能 |

### 0.3 一句话总结

> **DeepSpeed/FSDP 切"状态"，Megatron 切"模型"**——前者更通用（不改模型代码），后者更极致（要重写模型为 TP/PP 版）。**训练超大模型（>30B）通常两者结合**。

### 0.4 学到什么程度算"够用"？

| 级别 | 能力标志 |
|:--|:--|
| **G1 认知** | 说清 DP/TP/PP、ZeRO-1/2/3 |
| **G2 会用** | 用 FSDP 训通 7B，用 DeepSpeed 训通 30B |
| **G3 会调** | 会调 ZeRO stage、activation checkpoint、grad accum |
| **G4 会造** | 用 Megatron 训 100B+，理解 pipeline schedule |

## 1. 五种并行策略：DP / TP / PP / EP / SP

### 1.1 一张图看懂

```
                    数据 Batch
                       ↓
┌──────────────────────────────────────────────────┐
│  DP (数据并行)：每卡拿部分 batch，模型复制         │
│  TP (张量并行)：单个矩阵横切/竖切到多卡            │
│  PP (流水并行)：不同层放不同卡（前向后向流水化）    │
│  EP (专家并行)：MoE 的专家分卡                    │
│  SP (序列并行)：seq_len 维切分（LayerNorm/Dropout） │
└──────────────────────────────────────────────────┘
```

### 1.2 五种并行详解

#### ① DP（Data Parallel，数据并行）

- **切什么**：batch；
- **通信**：AllReduce 梯度；
- **典型**：`torch.nn.parallel.DDP`。

```
GPU 0: 模型完整 + batch[0:32]
GPU 1: 模型完整 + batch[32:64]
GPU 2: 模型完整 + batch[64:96]
GPU 3: 模型完整 + batch[96:128]
        ↓ backward
      AllReduce 梯度
        ↓
      每卡 optimizer.step()
```

**优点**：零改动、最简单；**缺点**：模型必须能放进单卡。

#### ② TP（Tensor Parallel，张量并行）

- **切什么**：单个 Linear/Attention 里的矩阵；
- **通信**：AllReduce / AllGather（forward + backward 每层都要）；
- **典型**：Megatron-LM 的 `ColumnParallelLinear` / `RowParallelLinear`。

```
一个 [4096 × 4096] 的 Linear 拆到 4 卡：
GPU 0: [4096 × 1024]
GPU 1: [4096 × 1024]
GPU 2: [4096 × 1024]
GPU 3: [4096 × 1024]
        ↓ 各自算 output 分块
       AllGather 得到完整 output
```

**优点**：单层能拆得很大；**缺点**：**必须机内 NVLink**（跨机太慢）。

#### ③ PP（Pipeline Parallel，流水并行）

- **切什么**：模型层（layer）；
- **通信**：P2P（点对点）传 activation；
- **典型**：GPipe / 1F1B schedule。

```
GPU 0: Layer 0~7
GPU 1: Layer 8~15
GPU 2: Layer 16~23
GPU 3: Layer 24~31
         ↓ activation 顺流而下
         ↑ gradient 逆流而上
```

**优点**：可以跨机；**缺点**：**bubble 问题**（首尾卡空闲）。

#### ④ EP（Expert Parallel，专家并行）

- **切什么**：MoE 的 experts；
- **通信**：All-to-All 分发 token；
- **典型**：Mixtral、DeepSeek-MoE。

```
Router 决定：token i 去 expert j
    ↓
All-to-All：把 token 送到对应 GPU
    ↓
每个 GPU 用自己的 expert 处理
    ↓
All-to-All：结果送回原 GPU
```

#### ⑤ SP（Sequence Parallel，序列并行）

- **切什么**：LayerNorm / Dropout 沿 seq_len 维切分；
- **动机**：这些操作的 activation 显存很大；
- **典型**：Megatron 3D 并行的补充。

### 1.3 通信量对比

| 并行 | 每层通信量 | 每 step 次数 |
|:--|:--|:--|
| DP | O(参数量) | 1（backward 末尾）|
| TP | O(activation 大小) | 每层 2~4 次 |
| PP | O(activation 大小) | 每 stage 边界 1 次 |
| EP | O(token × expert 覆盖率) | 每 MoE 层 2 次 |

**结论**：**TP 通信最重、DP 最轻**——所以 TP 必须走 NVLink，DP 可以跨机。

## 2. ZeRO：DeepSpeed 的核心

### 2.1 ZeRO 三个阶段

**观察**：DDP 里每卡都有完整的 params / grads / optim states —— **重复存了 N 份，太浪费！**

**ZeRO 就是逐步把这些"沉重的三兄弟"切分到不同卡**：

| Stage | 切分对象 | 显存节省 | 通信开销 |
|:--:|:--|:--|:--|
| **ZeRO-1** | 优化器状态 | 4× | 与 DDP 相当 |
| **ZeRO-2** | + 梯度 | 8× | 与 DDP 相当 |
| **ZeRO-3** | + 参数 | **N×**（N = GPU 数）| 增加 1.5× |

### 2.2 ZeRO-3 工作流程

```
每卡只存 1/N 的参数：
    GPU 0: params[0:P/N]
    GPU 1: params[P/N:2P/N]
    ...

Forward 时：
    需要哪层 → AllGather 参数 → 算完 → 立刻释放
    
Backward 时：
    需要哪层 → AllGather 参数 → 算梯度 → ReduceScatter 梯度 → 释放参数
    
每卡最终只留 1/N 的 grad 和 1/N 的 optim state
```

**代价**：多了 `AllGather` 参数的通信；**收益**：显存降到 1/N。

### 2.3 DeepSpeed ZeRO 配置示例

```json
{
  "train_batch_size": 128,
  "fp16": {"enabled": true},
  "zero_optimization": {
    "stage": 3,
    "offload_optimizer": {"device": "cpu"},
    "offload_param": {"device": "nvme"},
    "overlap_comm": true,
    "reduce_bucket_size": 5e8
  }
}
```

**Offload 是 ZeRO 的另一大杀器**：把优化器状态放 CPU / NVMe，进一步省显存（代价：慢 30%~100%）。

## 3. FSDP：PyTorch 原生的 ZeRO

### 3.1 FSDP = ZeRO-3

**PyTorch 官方 2021 年推出，本质就是 ZeRO-3**，API 更 Pythonic：

```python
from torch.distributed.fsdp import FullyShardedDataParallel as FSDP

model = MyLLM()
model = FSDP(model)   # 就这一行！
```

### 3.2 关键参数

```python
from torch.distributed.fsdp import (
    FullyShardedDataParallel as FSDP,
    ShardingStrategy,
    MixedPrecision,
    BackwardPrefetch,
)

model = FSDP(
    model,
    sharding_strategy=ShardingStrategy.FULL_SHARD,   # ZeRO-3
    # 也可选 SHARD_GRAD_OP (ZeRO-2), HYBRID_SHARD (机内 ZeRO-3 + 跨机 DP)
    mixed_precision=MixedPrecision(
        param_dtype=torch.bfloat16,
        reduce_dtype=torch.bfloat16,
        buffer_dtype=torch.bfloat16,
    ),
    backward_prefetch=BackwardPrefetch.BACKWARD_PRE,   # 预取参数
    auto_wrap_policy=size_based_auto_wrap_policy,
    device_id=torch.cuda.current_device(),
)
```

### 3.3 FSDP2：新一代 API（PyTorch 2.4+）

```python
from torch.distributed._composable.fsdp import fully_shard

for layer in model.layers:
    fully_shard(layer)      # 每层单独 shard
fully_shard(model)          # 顶层再 shard 一次
```

**优点**：模块级控制、更好和 `torch.compile` / `TP` 组合。**是未来主推方向**。

## 4. Megatron-LM：3D 并行的教科书

### 4.1 Megatron 的三大贡献

1. **Tensor Parallel**（2019）：ColumnParallelLinear + RowParallelLinear；
2. **Pipeline Parallel + 1F1B schedule**（2021）：缓解 bubble；
3. **Sequence Parallel**（2022）：进一步省 activation 显存。

### 4.2 3D 并行示意

```
假设 8 卡训练：DP=2, TP=2, PP=2

              PP Stage 0        PP Stage 1
            ┌───────────┐    ┌───────────┐
DP Rank 0:  │ GPU0 GPU1 │───▶│ GPU4 GPU5 │
            │ (TP0)(TP1)│    │ (TP0)(TP1)│
            └───────────┘    └───────────┘
            ┌───────────┐    ┌───────────┐
DP Rank 1:  │ GPU2 GPU3 │───▶│ GPU6 GPU7 │
            └───────────┘    └───────────┘
```

**通信拓扑**：
- **TP 组**：机内 NVLink（0-1, 2-3, 4-5, 6-7）；
- **DP 组**：AllReduce 梯度（0-2, 1-3, ...）；
- **PP 组**：P2P activation（0→4, 1→5, ...）。

### 4.3 1F1B 调度：Pipeline 的关键优化

**朴素 GPipe**：所有 forward 完成才开始 backward → bubble 很大。

**1F1B（One-Forward One-Backward）**：一旦一个 micro-batch 完成 forward，就立即开始 backward：

```
GPU 0: F0 F1 F2 F3 B0 F4 B1 F5 B2 B3 B4 B5
GPU 1:    F0 F1 F2 B0 F3 B1 F4 B2 F5 B3 B4 B5
                    ↑ backward 提前开始
```

**收益**：bubble 从 O(P) 降到 O(P/N_micro)。

### 4.4 Megatron 里写 Column/Row Parallel Linear

```python
# Column-parallel: 按输出维度切
class ColumnParallelLinear(nn.Module):
    def __init__(self, in_dim, out_dim, tp_size):
        self.weight = Parameter(torch.empty(out_dim // tp_size, in_dim))
        # 每卡只持有 [out/tp, in] 的部分权重

    def forward(self, x):
        # 输入 x 复制到每卡
        y_partial = F.linear(x, self.weight)   # [batch, out/tp]
        # 不 AllGather，直接返回（作为下一层的输入）
        return y_partial

# Row-parallel: 按输入维度切
class RowParallelLinear(nn.Module):
    def __init__(self, in_dim, out_dim, tp_size):
        self.weight = Parameter(torch.empty(out_dim, in_dim // tp_size))

    def forward(self, x_partial):   # 每卡持有 [batch, in/tp]
        y_partial = F.linear(x_partial, self.weight)   # [batch, out]
        y = all_reduce(y_partial)   # ← 求和得到完整 output
        return y
```

**关键设计**：Attention/MLP 里 **Column + Row 成对**使用，Column 输出直接喂给 Row，中间不用 AllGather，只在末尾 AllReduce。

## 5. 三剑客对比与选型

### 5.1 对比表

| 维度 | DeepSpeed | FSDP | Megatron-LM |
|:--|:--|:--|:--|
| **出品方** | 微软 | PyTorch (Meta) | NVIDIA |
| **核心技术** | ZeRO 1/2/3 + Offload | ZeRO-3 (原生) | TP + PP + SP |
| **改模型代码** | ❌ 无需 | ❌ 无需 | ✅ 大改（换 Linear）|
| **通信开销** | 中 | 中 | 高（TP）|
| **显存优化** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **性能上限** | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **适用规模** | 7B~100B | 7B~70B | 30B~千亿 |
| **易用性** | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |
| **CPU/NVMe Offload** | ✅ | ⚠️ 有限 | ❌ |

### 5.2 选型决策树

```
你要训多大模型？
    │
    ├─ ≤ 7B
    │    ├─ 想省事 → DDP（如果单卡放得下）
    │    └─ 单卡放不下 → FSDP（PyTorch 原生，最简单）
    │
    ├─ 7B ~ 70B
    │    ├─ 追求易用 → FSDP + activation checkpoint
    │    ├─ 追求省显存 → DeepSpeed ZeRO-3 + Offload
    │    └─ 追求性能 → Megatron TP=2/4 + FSDP
    │
    └─ 70B+
         └─ Megatron 3D 并行（TP + PP + DP）
            + FSDP/DeepSpeed 补 ZeRO
```

## 6. 显存 & 通信量估算公式

### 6.1 显存估算（fp16 训练，AdamW）

**每参数需要显存**：
- FP16 params: 2 bytes
- FP16 grads: 2 bytes
- FP32 master params: 4 bytes
- FP32 Adam m: 4 bytes
- FP32 Adam v: 4 bytes
- **合计: 16 bytes/param**

**7B 模型**：7e9 × 16 = **112 GB**（不含 activation）。

### 6.2 ZeRO 后的显存

| 并行 | 每卡显存（不含 activation）|
|:--|:--|
| DDP | 16P |
| ZeRO-1 | 4P + 12P/N |
| ZeRO-2 | 2P + 14P/N |
| ZeRO-3 | 16P/N |
| Megatron TP | 16P/TP + 通信 buffer |

其中 P = 参数量、N = DP 组大小。

### 6.3 Activation 显存与 checkpoint

**Activation** 大小 ≈ `layers × batch × seq_len × hidden × 4` bytes。

**7B, bsz=1, seq=4K**：32 × 1 × 4096 × 4096 × 4 ≈ **2 GB**（fp16 时减半）。

**Activation checkpointing**：只保存部分 activation，反向时重算，显存降 **√L 倍**，代价 25% 算力。

```python
from torch.utils.checkpoint import checkpoint

def forward(self, x):
    for layer in self.layers:
        x = checkpoint(layer, x)   # ← 重算
    return x
```

## 7. 常见组合拳：Megatron + DeepSpeed、FSDP + TP

### 7.1 Megatron-DeepSpeed（微软 + NVIDIA）

- Megatron 提供 TP + PP；
- DeepSpeed 提供 ZeRO + Offload；
- **训练 GPT-3 175B 的官方配置**：TP=8, PP=16, DP=... + ZeRO-1。

### 7.2 FSDP + TP（PyTorch 官方新方向）

- 用 `DTensor` 表达 TP；
- 与 FSDP2 组合成 2D 并行；
- **Meta 训练 Llama 3 的官方栈**。

### 7.3 组合原则

| 组合 | 优势 |
|:--|:--|
| DP + ZeRO/FSDP | 最省显存（切状态）|
| DP + TP | 单层能拆大（切模型）|
| DP + PP | 跨机友好（切层）|
| DP + TP + PP | 3D 并行，训超大模型必用 |
| DP + TP + PP + ZeRO-1 | 4D，175B+ 标配 |

## 8. 端到端示例：7B 模型 8 卡训练

### 8.1 用 FSDP（最简单）

```bash
torchrun --nproc_per_node=8 train.py
```

```python
# train.py
import torch
from torch.distributed.fsdp import FullyShardedDataParallel as FSDP, MixedPrecision
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-hf")
model = FSDP(
    model,
    mixed_precision=MixedPrecision(param_dtype=torch.bfloat16),
    device_id=torch.cuda.current_device(),
)

optimizer = torch.optim.AdamW(model.parameters(), lr=1e-5)

for batch in dataloader:
    with torch.autocast(device_type='cuda', dtype=torch.bfloat16):
        loss = model(**batch).loss
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()
```

### 8.2 用 DeepSpeed（更省显存）

```bash
deepspeed --num_gpus=8 train.py --deepspeed ds_config.json
```

```python
import deepspeed
model, optimizer, _, _ = deepspeed.initialize(
    model=model,
    model_parameters=model.parameters(),
    config="ds_config.json",
)

for batch in dataloader:
    loss = model(**batch).loss
    model.backward(loss)
    model.step()
```
