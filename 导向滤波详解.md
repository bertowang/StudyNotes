## 0. 导向滤波要解决什么问题？

先看一个几乎每个搞图像的人都遇到过的两难问题：

> 我想去掉图像里的噪点/纹理（**平滑**），但又不想把物体的**边缘**也一起模糊掉。

**方案对比**：

| 滤波方式 | 是否保边 | 复杂度 | GPU 友好 | 备注 |
|---------|---------|--------|---------|------|
| 均值/高斯模糊 | ❌ 边被糊 | O(N·r²) 或 O(N)（可分离）| ✅ | 简单粗暴 |
| 中值滤波 | ⚠️ 部分保 | O(N·r² log r) | ⚠️ | 抗椒盐噪声好 |
| **双边滤波** | ✅ 保边 | O(N·r²)，加速版 O(N) 但常数大 | ⚠️ 分支多 | 经典保边滤波 |
| **导向滤波** | ✅ 保边 | **严格 O(N)**（不依赖 r） | ✅✅ 只用 box filter | **本文主角** |
| 深度学习去噪 | ✅ 极好 | 需 GPU 推理 | ✅ | 依赖训练数据 |

**导向滤波的独特优势**：

1. **保边**：效果接近双边滤波；
2. **快**：复杂度和窗口半径 `r` **无关**（因为只用 box filter）；
3. **闭式解**：没有迭代，一次算完；
4. **梯度保持**：不像双边滤波会产生"梯度反转"伪影；
5. **应用广**：抠图、去雾、HDR、上采样、磨皮 …… 一个滤波器打天下。

这正是它 2010 年一发表就成为工业界标配、现在 OpenCV/GIMP/Photoshop 里都有实现的原因。

## 1. 直观理解：为什么普通模糊会毁掉边缘

考虑一张 1D 信号（想象一行像素）：

```
输入 p：黑左白右，中间夹一点噪声
   ┌ ─ ─ ─ ─ ─ ┐┌ ─ ─ ─ ─ ─ ┐
0.1  0.05  0.12   0.9  0.88  0.95     ← 噪声在两块平坦区里
─────────────────────────────
       黑            白
       └── 边缘在这里 ──┘
```

**高斯模糊**：把边缘附近的黑白都平均起来：

```
输出：0.1  0.15  0.35   0.65  0.85  0.9
                     ↑↑↑
                     边被糊成灰色渐变了！
```

**导向滤波的思路**：

> 与其"盲目地"取邻域平均，不如**先看看这块区域本身有没有边**——如果没有边（都差不多），就大胆平均；**如果有边，就少动它，让输出保持这条边**。

导向滤波用一个引导图 `I`（可以就是输入 `p` 自己）来判断"哪里是边"。它的核心断言是：

> **在一个不跨过边的小窗口里，输出 `q` 与引导图 `I` 呈线性关系 `q = a·I + b`。**

只要这条断言成立，边就能被完整保住——因为线性变换不会改变梯度的位置，只会缩放它。

### 1.1 为什么"线性变换不会改变梯度的位置，只会缩放它"？

上面这句话是导向滤波保边能力的**核心逻辑**，值得掰开揉碎讲清楚。我们分四步：

**① 先明确：什么叫"边"？**

在图像里，**"边"就是像素值发生剧烈变化的地方**，用数学语言说就是**梯度 `∇I` 在那里很大**。

- **平坦区**（一片天空、一堵白墙）：相邻像素几乎相等 ⇒ `∇I ≈ 0`；
- **边缘处**（黑白交界、物体轮廓）：相邻像素跳变很大 ⇒ `|∇I|` 很大；
- **边的"位置"**：就是 `|∇I|` 取到极大值的那些像素。

所以**"保住边"的严格含义**是：**输出图 q 在这些位置也要出现同样大的梯度，且位置不能偏移**。

**② 对线性关系两边同时求梯度**

假设窗口内 `q = a·I + b`（其中 `a`、`b` 是这个窗口共用的两个常数），对两侧同时求梯度（离散图像上就是差分）：

```
    q       =  a · I  +  b
    ↓ 求梯度       ↓         ↓
    ∇q      =  a · ∇I +  0
              └───────┘
              b 是常数，梯度为 0
```

结论：

```
    ┌─────────────────────┐
    │     ∇q  =  a · ∇I   │
    └─────────────────────┘
```

这就是"**线性变换只缩放梯度**"的严格表述。它告诉我们两件事：

- **方向**：`∇q` 与 `∇I` 同向（`a > 0` 时）或反向（`a < 0` 时）——**梯度的方向被完整继承**；
- **大小**：`∇q` 是 `∇I` 的 `a` 倍——只是**幅值**被缩放了。

**③ 为什么"位置不变"？——一维直觉**

想象一段一维信号 `I`：左半段是 0.1，右半段是 0.9，边就在中间那个像素。

假设窗口内取 `a = 0.8`, `b = 0.05`，那么：

```
    位置：   1     2     3   │   4     5     6
    I：    0.1   0.1   0.1  │  0.9   0.9   0.9
    q：    0.13  0.13  0.13 │  0.77  0.77  0.77   （q = 0.8·I + 0.05）
    ∇I：    0     0     0.8 │   0     0     0     （在位置 3→4 之间）
    ∇q：    0     0     0.64│   0     0     0     （梯度乘了 0.8 倍）
                            └─位置完全不变─┘
```

**关键观察**：`∇I` 在哪个位置爆发，`∇q` 就在**完全同一个位置**爆发；平坦区两边照样平坦——**边的位置一个像素也没有偏移**，只是"高度"被缩放了。

对比一下如果用高斯模糊会怎样：

```
    高斯模糊后 q：0.13  0.20  0.35 │ 0.55  0.70  0.77
                       ↑     ↑     ↑     ↑
                       原本平坦的 2、3、4、5 位置全都开始"过渡"了
                       ⇒ 边被"抹开"到多个像素上，位置模糊、幅度也降了
```

——这就是"线性变换保边"和"卷积模糊毁边"的本质区别。

**④ 平坦区与边缘区的自动切换**

线性模型 `q = a·I + b` 的美妙之处，还在于 `a` 会**根据窗口内容自动调整**（后面第 3 章会推出闭式解）：

| 窗口类型 | 引导图特征 | 求解得到的 `a` | 输出 `q` 的效果 |
|:--|:--|:--|:--|
| **平坦区**（无边）| `∇I ≈ 0`，`Var(I) ≈ 0` | `a → 0` | `q ≈ b ≈ p̄`（均值），**强平滑去噪** |
| **边缘区**（有边）| `Var(I)` 很大 | `a → 1` | `q ≈ I`，**原样保留边** |
| **弱纹理区** | `Var(I)` 中等 | `0 < a < 1` | 部分平滑、部分保留 |

也就是说：**同一个公式 `q = aI + b`**，在平坦区自动变成"取均值"（平滑），在边缘区自动变成"复制引导图"（保边）——**一套模型两种行为，全靠数据本身决定**。这正是导向滤波比高斯模糊/双边滤波都优雅的地方。

> **一句话记忆**：`∇q = a·∇I` ⇒ 边的**位置**由 `∇I` 决定（不动），边的**强弱**由 `a` 决定（可缩放）。位置由引导图独占，滤波器只能碰"强度"——所以边永远不会被搬走或抹掉。

### 1.2 什么是"引导图 I"？——一个通俗解释

上面反复出现一个关键角色：**引导图 `I`**（Guidance Image）。它是导向滤波区别于所有传统滤波器的**灵魂**。这一小节用大白话把它讲透。

**① 一句话定义**

> **引导图 = 一张"告诉滤波器哪里是边、哪里是平坦区"的参考图。**

滤波器自己不知道哪里该模糊、哪里该保留，它需要一个"顾问"来指路。这个顾问，就是引导图 `I`。

**② 生活化比喻：描红本**

想象小朋友学写字用的**描红本**：

| 角色 | 描红本场景 | 导向滤波中的对应 |
|:--|:--|:--|
| 底稿（浅灰色的字） | 印在纸上的模板，告诉小朋友"笔画在这里" | **引导图 `I`**（告诉滤波器"边在这里"） |
| 小朋友的笔 | 沿着模板一笔笔描 | **输入图 `p`**（要被滤波的原始数据） |
| 描完的作品 | 结构跟模板一致，但用的是自己的墨水 | **输出图 `q`**（结构像 `I`、内容像 `p`） |

**核心机制**：输出 `q` 的**结构（边缘位置）向 `I` 看齐**，但**亮度/数值向 `p` 看齐**。

**③ 三种典型用法**

引导图具体是什么图，取决于你想让滤波器"参考谁的边"：

**用法 1：`I = p`（自导向，最常见）**

引导图就是输入图自己。相当于"我看着自己去噪"——**用自身结构指导自身平滑**。这就是常规的保边去噪、磨皮场景。

```
    输入 p ──┬──> 当输入
             │
             └──> 当引导 I     （同一张图，两个角色）
```

**用法 2：`I ≠ p`（联合滤波）**

引导图和输入图是**两张不同的图**，但描述的是同一个场景。典型例子：

- **深度图上采样**：`p` = 低分辨率、有噪声的深度图；`I` = 高清彩色照片。
    - 彩色图的边缘清晰，用它指导深度图 → 得到边缘对齐的高清深度图。
- **闪光/无闪光去噪**：`p` = 无闪光的暗光照片（噪声大）；`I` = 同一场景的闪光照片（清晰但色调怪）。
    - 用闪光图的结构指导无闪光图去噪 → 保留自然色调 + 干净结构。
- **抠图 Alpha 优化**：`p` = 粗糙的 alpha 蒙版；`I` = 原始彩色照片。
    - 用照片的头发丝细节指导蒙版 → 蒙版边缘贴合头发。

**用法 3：`I` 是特殊构造图**

比如去雾算法里，`I` 是彩色原图，`p` 是粗略估计的透射率图——用清晰彩色图去优化透射率的边缘。

**④ 引导图必须满足什么条件？**

不是随便一张图都能当引导图，它必须满足：

1. **尺寸和 `p` 一样**（或者能对齐——快速导向滤波允许先下采样）；
2. **边缘位置和 `p` 想要的输出一致**（否则会把错误的边"教"给输出）；
3. **本身噪声不能太大**（噪声大 = 假边多，会误导滤波器）。

**⑤ 一张图看清 I / p / q 三者关系**

```
       ┌──────────┐         ┌──────────┐
       │  输入 p  │         │  引导 I  │
       │ (待滤波) │         │(结构参考)│
       └────┬─────┘         └────┬─────┘
            │                    │
            │  "帮我平滑，       │ "边在哪儿？
            │   但保住 I 的边"   │  我告诉你"
            │                    │
            └────────┬───────────┘
                     ↓
              ┌──────────────┐
              │ 导向滤波器   │
              │ q = a·I + b  │
              └──────┬───────┘
                     ↓
              ┌──────────────┐
              │   输出 q     │
              │ 结构≈I，值≈p │
              └──────────────┘
```

**⑥ 一句话总结**

> **`p` 提供"内容"，`I` 提供"结构"，`q` 兼得两者。**
> 当 `I = p` 时，就是"用自己指导自己"的保边去噪；当 `I ≠ p` 时，就能玩出联合上采样、去雾、抠图等各种花样——**一个引导图，让同一个滤波器变身十八般武艺**。

## 2. 核心思想：局部线性模型

### 2.1 一句话定义

**导向滤波假设**：在**每一个以像素 k 为中心的小窗口 `ω_k`**（半径 `r`）内，输出图像 `q` 是引导图 `I` 的**线性函数**：

```
    ┌────────────────────────────────────────┐
    │   q_i  =  a_k · I_i  +  b_k    ∀i∈ω_k  │
    └────────────────────────────────────────┘
```

- `I`：**引导图**（Guidance），提供"边缘信息"；
- `p`：**输入图**（要被滤波的），可能带噪声；
- `q`：**输出图**（滤波结果）；
- `(a_k, b_k)`：这个窗口内的局部线性系数——**每个窗口有自己的一组 (a, b)**；
- `r`：窗口半径；`ε`：正则化系数（下面会说）。

**当 `I = p` 时**（自导向），就是最常见的"保边平滑"用法。当 `I ≠ p` 时，可以做很多有意思的事情——比如 `I` 是彩色图、`p` 是深度图，就是**联合上采样**。

### 2.2 为什么"线性"就能保边

**关键性质**：如果 `q = aI + b`，两边求梯度：

```
∇q = a · ∇I
```

**梯度只是被缩放了 `a` 倍，位置和方向都不变**。也就是说：

- `I` 里的边 → `q` 里同位置也有边；
- `I` 里的平坦区（`∇I ≈ 0`）→ `q` 也平坦（`∇q ≈ 0`）；
- `a` 越小，`q` 越接近常数（越平滑）；`a` 越大，`q` 越"跟随"`I` 的结构。

我们**希望**：

- 在**平坦区**（`I` 方差小）：`a → 0`，`b → 均值` ⇒ 强平滑；
- 在**边缘区**（`I` 方差大）：`a → 1`，`b → 0` ⇒ 保留结构。

下面的推导会自动得出这个结果——这就是导向滤波的魅力。

---

## 3. 数学推导：从代价函数到闭式解

### 3.1 代价函数

我们希望输出 `q = aI + b` 尽可能接近输入 `p`（去噪但别偏太多），同时**避免 a 过大**（否则会放大噪声）。在窗口 `ω_k` 里定义：

```
    E(a_k, b_k)  =  Σ_{i∈ω_k}  [ (a_k · I_i + b_k - p_i)²  +  ε · a_k² ]
                                └─── 数据项 ───┘         └─正则项─┘
```

- **数据项**：让 `q_i = aI_i + b` 逼近 `p_i`；
- **正则项 `ε·a²`**：抑制 a 过大——`ε` 越大，越倾向于把 `a` 压向 0（更平滑）。

这是**岭回归（Ridge Regression）**的标准形式。

### 3.2 对 (a, b) 求偏导 = 0

令 `n = |ω_k|` 为窗口内像素数。

**对 `b_k` 求偏导**：

```
∂E/∂b_k = 2 Σ (a_k I_i + b_k - p_i) = 0
       ⇒  a_k · Σ I_i + n·b_k = Σ p_i
       ⇒  b_k = p̄_k - a_k · μ_k          ······ (⋆)
```

其中 `μ_k = (1/n) Σ I_i` 是引导图在窗口内的均值，`p̄_k = (1/n) Σ p_i` 是输入图的均值。

**对 `a_k` 求偏导**：

```
∂E/∂a_k = 2 Σ I_i (a_k I_i + b_k - p_i) + 2ε a_k = 0
       ⇒  a_k Σ I_i² + b_k Σ I_i - Σ I_i p_i + ε a_k = 0
```

代入 (⋆)：`b_k = p̄_k - a_k μ_k`：

```
   a_k Σ I_i² + (p̄_k - a_k μ_k)·Σ I_i - Σ I_i p_i + ε a_k = 0
   a_k [ Σ I_i² - μ_k · Σ I_i + ε ] = Σ I_i p_i - p̄_k · Σ I_i
```

用**方差 / 协方差**改写右边（这是标准恒等式）：

```
Σ I_i² - μ_k · Σ I_i = n · σ_k²                     (σ² 是方差)
Σ I_i p_i - p̄_k · Σ I_i = n · cov_k(I, p)            (协方差)
```

代入并两边除以 `n`：

```
a_k · (σ_k² + ε/n) = cov_k(I, p)
```

**吸收常数**（教科书里一般把 `ε/n` 直接写成 `ε`，因为 ε 本就是可调正则参数）：

### 3.3 关键结果：a、b 的闭式解

```
    ┌──────────────────────────────────────────────┐
    │                                              │
    │         cov_k(I, p)                          │
    │  a_k = ────────────                          │
    │         σ_k² + ε                             │
    │                                              │
    │  b_k = p̄_k - a_k · μ_k                      │
    │                                              │
    └──────────────────────────────────────────────┘
```

**几何/统计含义**：

- **平坦区**：`σ_k²` 很小 ⇒ `a_k ≈ cov/ε` 也很小 ⇒ `q ≈ b_k ≈ p̄_k`（**取邻域平均，强平滑**）；
- **边缘区**：`σ_k²` 很大 ⇒ 分母 σ² 主导 ⇒ `a_k → cov/σ² → 1`（当 I=p 时），`b_k → 0` ⇒ `q ≈ I`（**保留原样**）；
- **`ε` 是"平滑门槛"**：只有当 `σ² > ε` 时才认定"这里有边、别乱平滑"。

**当 I = p 时**（自导向情形）：

```
cov_k(p, p) = σ_k²                          (方差就是自协方差)

a_k = σ_k² / (σ_k² + ε)                    ← 越接近 1 越保边
b_k = (1 - a_k) · μ_k
```

漂亮！`a_k ∈ [0, 1)` 天然是一个"边缘置信度"。

### 3.4 输出的最终形式：均值化

我们对**每个窗口 `ω_k`** 都算了一组 `(a_k, b_k)`。但同一个像素 `i` 会同时被多个窗口覆盖（覆盖它的窗口数 = `(2r+1)²`），每个窗口都会给它一个"预测值"。

**做法**：把所有覆盖 i 的 (a_k, b_k) **平均**起来：

```
    ā_i = (1/n) Σ_{k: i∈ω_k} a_k
    b̄_i = (1/n) Σ_{k: i∈ω_k} b_k
```

这**等价于**对整幅 a、b 图像做一次 **box filter（均值滤波）**。最后：

```
    ┌────────────────────────────────┐
    │    q_i  =  ā_i · I_i  +  b̄_i   │
    └────────────────────────────────┘
```

这一步"再平均一次"至关重要——它让相邻窗口的系数平滑过渡，避免出现"块效应"。

## 4. 完整算法（5 步）

给定引导图 `I`、输入图 `p`、半径 `r`、正则系数 `ε`，box_r 表示半径 r 的均值滤波：

```
Step 1: mean_I     = box_r(I)              μ_k
        mean_p     = box_r(p)              p̄_k
        mean_II    = box_r(I .* I)
        mean_Ip    = box_r(I .* p)

Step 2: var_I      = mean_II - mean_I .* mean_I           σ_k²
        cov_Ip     = mean_Ip - mean_I .* mean_p           cov_k(I, p)

Step 3: a          = cov_Ip ./ (var_I + ε)
        b          = mean_p - a .* mean_I

Step 4: mean_a     = box_r(a)              ā_i
        mean_b     = box_r(b)              b̄_i

Step 5: q          = mean_a .* I + mean_b               输出
```

**总共 6 次 box filter + 少量逐像素运算**。box filter 用积分图或可分离滤波都是 `O(N)`——**总复杂度 `O(N)`，与 r 无关**！

---

## 5. 参数意义：r 和 ε

| 参数 | 含义 | 大 → | 小 → |
|------|------|------|------|
| `r` | 窗口半径 | 平滑范围更大、能跨更长距离聚合信息 | 只在很小的邻域内平均 |
| `ε` | 正则化系数（"边缘门槛"） | 更强平滑，即使有一定方差的区域也被抹平 | 只有极强梯度才被保留（近乎不动）|

**经验值**：

- 磨皮/去噪：`r = 4~16`，`ε = (0.1 * 255)² ≈ 650`（8bit 图）或 `ε = 0.01²` ~ `0.1²`（[0,1] 归一化图）；
- HDR 色调映射：`r` 较大（如 15），`ε` 较大；
- 抠图 / 深度图上采样：`r` 中等，`ε` 很小（保结构）。

## 6. 完整实例：手算一张 5×5 图像

用一个**可以纸笔完全手算**的最小例子把 5 步全部走完。

### 6.1 输入图像

一张 5×5 图像，左半黑右半白，中间夹一列**边缘**。为演示"保边"效果，我们让左半有点噪声：

```
       x=0   x=1   x=2   x=3   x=4
     ┌─────┬─────┬─────┬─────┬─────┐
y=0  │ 0.0 │ 0.2 │ 0.1 │ 0.9 │ 1.0 │
     ├─────┼─────┼─────┼─────┼─────┤
y=1  │ 0.1 │ 0.0 │ 0.2 │ 1.0 │ 0.9 │
     ├─────┼─────┼─────┼─────┼─────┤
y=2  │ 0.0 │ 0.1 │ 0.1 │ 0.9 │ 1.0 │       ← 中心行
     ├─────┼─────┼─────┼─────┼─────┤
y=3  │ 0.2 │ 0.0 │ 0.1 │ 1.0 │ 0.9 │
     ├─────┼─────┼─────┼─────┼─────┤
y=4  │ 0.0 │ 0.1 │ 0.2 │ 0.9 │ 1.0 │
     └─────┴─────┴─────┴─────┴─────┘
       └────左半（黑，有噪）────┘  └边界┘  └右半（白）┘
```

设 `I = p`（自导向），是**去噪**任务。

### 6.2 参数设置

```
r = 1     (3×3 窗口，n = 9)
ε = 0.01
```

### 6.3 计算局部均值 μ 和 p̄

我们**只算中心行 y=2 的三个内部像素**（x=1, 2, 3），边界处理用"缩窗口"（不越界）。因 `I = p`，`μ_k = p̄_k`。

**像素 (2, 1)**——中心 3×3 窗口是 `x ∈ [0,2], y ∈ [1,3]`：

```
窗口值：
 0.1  0.0  0.2
 0.0  0.1  0.1
 0.2  0.0  0.1

μ = (0.1+0.0+0.2 + 0.0+0.1+0.1 + 0.2+0.0+0.1)/9
  = 0.8/9 ≈ 0.0889
```

**像素 (2, 2)**——中心 3×3 窗口 `x ∈ [1,3], y ∈ [1,3]`（**跨边界的关键窗口**）：

```
窗口值：
 0.0  0.2  1.0
 0.1  0.1  0.9
 0.0  0.1  1.0

μ = (0.0+0.2+1.0 + 0.1+0.1+0.9 + 0.0+0.1+1.0)/9
  = 3.4/9 ≈ 0.3778
```

**像素 (2, 3)**——中心 3×3 窗口 `x ∈ [2,4], y ∈ [1,3]`：

```
窗口值：
 0.2  1.0  0.9
 0.1  0.9  1.0
 0.1  1.0  0.9

μ = (0.2+1.0+0.9 + 0.1+0.9+1.0 + 0.1+1.0+0.9)/9
  = 6.1/9 ≈ 0.6778
```

### 6.4 计算局部方差 σ² 和协方差

因 `I = p`：`cov_k(I, p) = σ_k²`。

**像素 (2, 1) 的 σ²**：所有值都在 [0, 0.2] 之间，方差很小。

```
E[I²] = (0.01+0+0.04 + 0+0.01+0.01 + 0.04+0+0.01)/9 = 0.12/9 ≈ 0.01333
μ² = 0.0889² ≈ 0.00790
σ² ≈ 0.01333 - 0.00790 = 0.00543         ← 很小，平坦区
```

**像素 (2, 2) 的 σ²**（跨边界窗口）：

```
所有 I² 值：
 0.00   0.04   1.00
 0.01   0.01   0.81
 0.00   0.01   1.00

E[I²] = (0.00+0.04+1.00 + 0.01+0.01+0.81 + 0.00+0.01+1.00)/9
      = 2.88/9 = 0.32
μ² = 0.3778² ≈ 0.1427
σ² ≈ 0.32 - 0.1427 = 0.1773               ← 很大，边缘区！
```

**像素 (2, 3) 的 σ²**：

```
所有 I² 值：
 0.04   1.00   0.81
 0.01   0.81   1.00
 0.01   1.00   0.81

E[I²] = 5.49/9 = 0.61
μ² = 0.6778² ≈ 0.4594
σ² ≈ 0.61 - 0.4594 = 0.1506
```

**等等——(2,3) 的 σ² 也不小啊？** 是的，因为它这个窗口也跨了一部分边界（左边有 0.2 和 0.1）。这在 `r=1` 的极小窗口下是必然现象。真实用 `r=8` 时边缘会更集中在 x=2 附近。

### 6.5 求 (a, b)

```
a_k = σ_k² / (σ_k² + ε)          (自导向)
b_k = (1 - a_k) · μ_k
```

| 像素 | σ² | a = σ²/(σ²+ε) | b = (1-a)·μ |
|------|-----|---------------|-------------|
| (2,1) 平坦区 | 0.00543 | 0.00543/0.01543 ≈ **0.352** | (1-0.352)·0.0889 ≈ 0.0576 |
| (2,2) 边缘 | 0.1773 | 0.1773/0.1873 ≈ **0.947** | (1-0.947)·0.3778 ≈ 0.0200 |
| (2,3) 半边 | 0.1506 | 0.1506/0.1606 ≈ **0.938** | (1-0.938)·0.6778 ≈ 0.0420 |

**观察**：

- 平坦区 `a ≈ 0.35`（较小，倾向于取均值 ≈ 0.06）；
- 边缘处 `a ≈ 0.95`（很大，几乎完全跟随 I）；
- 这就是"根据方差自适应决定平滑强度"的精髓！

### 6.6 求 (ā, b̄) 均值

对 `a`、`b` 图像再做一次 box filter（继续用 3×3）。为了继续能手算，我们**只算 (2, 2) 这个像素的最终输出**——需要它周围 3×3 的 `a` 和 `b` 值。

但我们只手算了 3 个像素的 (a, b)！在真实计算里所有像素都会有值。这里为演示，**我们用一个近似**：假设 (2, 2) 周围 8 邻居的 a、b 大致等于**它们所在区域的代表值**：

- 左侧邻居（属于平坦区）：`a ≈ 0.35, b ≈ 0.06`
- 右侧邻居（属于白平坦区）：`a ≈ 0.35, b ≈ 0.65`（对称推理）
- 中心 (2,2)：`a ≈ 0.947, b ≈ 0.020`

粗略估算 (2, 2) 处 `ā`：3 个"边缘型" a≈0.94 + 6 个"平坦型" a≈0.35：

```
ā(2,2) ≈ (3·0.947 + 6·0.35) / 9 ≈ (2.841 + 2.10)/9 ≈ 0.549
b̄(2,2) ≈ (3·0.020 + 3·0.06 + 3·0.65)/9 ≈ (0.06 + 0.18 + 1.95)/9 ≈ 0.243
```

（真实数值需要把 25 个像素的 a、b 全都算出来再滤一次，这里只做数量级演示）

### 6.7 得到输出 q

**像素 (2, 2) 的输入 `I(2,2) = 0.1`**（左半区的一个像素，注意坐标 x=2 属于边界，我们把它算作左半）：

```
q(2,2) = ā · I + b̄
       = 0.549 · 0.1 + 0.243
       ≈ 0.298
```

**如果换成高斯模糊**（比如同样 3×3 均值滤波）：

```
q_gauss(2,2) = μ(2,2) ≈ 0.378
```

### 6.8 结果分析

| 方法 | 输出 q(2,2) | 距离原值 0.1 |
|------|-----------|-------------|
| 原图 | 0.10 | 0 |
| 均值模糊 | 0.378 | 严重被右侧的白拉高 |
| **导向滤波** | 0.298 | 更接近原值，边界更清晰 |

虽然因为窗口太小、r=1 效果不够戏剧化，但方向已经清楚——**导向滤波在边缘处输出更接近本侧的原值，比均值滤波保边**。真实使用中一般 `r=8~16`，效果差异会非常明显。

**如果我们看纯平坦区**（比如 (0,0) 处，全是 0~0.2 的黑区）：

- 均值滤波和导向滤波都会得到 ~0.09 的平均值；
- 但**导向滤波会保留一部分噪声结构**（因为 a>0），只是**幅度被压小**（乘以 a<1）。这既减弱了噪声，又不会过度平滑得像塑料一样——这正是它比高斯"看起来更自然"的原因。

## 7. 工程实现要点

### 7.1 用积分图实现 O(N)

Box filter（对任意窗口的求和）可以用**积分图（Summed Area Table）**做到 `O(1)` 每个像素：

```
SAT(x, y) = Σ_{x'≤x, y'≤y} I(x', y')

窗口 [x1..x2, y1..y2] 之和
    = SAT(x2, y2) - SAT(x1-1, y2) - SAT(x2, y1-1) + SAT(x1-1, y1-1)
```

**只需 4 次查表和 3 次加减**——**与窗口大小 r 无关**！这就是导向滤波"**复杂度不依赖 r**"的技术根基。

### 7.2 OpenCV 代码示例

OpenCV 的 `ximgproc` 模块（contrib 包）直接提供了导向滤波：

```python
import cv2
import numpy as np

# 读入图像并归一化到 [0, 1]
img = cv2.imread("noisy.png").astype(np.float32) / 255.0

# 自导向：I = p
guided = cv2.ximgproc.guidedFilter(
    guide=img,           # 引导图 I
    src=img,             # 输入图 p
    radius=8,            # 窗口半径 r
    eps=0.02**2)         # 正则化 ε（0.02 对 [0,1] 图像很平滑）

cv2.imwrite("smooth.png", (guided * 255).astype(np.uint8))
```

**手写实现**（30 行搞定）：

```python
def guided_filter(I, p, r, eps):
    box = lambda x: cv2.boxFilter(x, -1, (2*r+1, 2*r+1))

    mean_I  = box(I)
    mean_p  = box(p)
    mean_II = box(I * I)
    mean_Ip = box(I * p)

    var_I   = mean_II - mean_I * mean_I
    cov_Ip  = mean_Ip - mean_I * mean_p

    a = cov_Ip / (var_I + eps)
    b = mean_p - a * mean_I

    mean_a = box(a)
    mean_b = box(b)

    return mean_a * I + mean_b
```

### 7.3 快速导向滤波（下采样加速）

对 4K 图像仍然嫌慢？何恺明后续发表了 **Fast Guided Filter**：

1. 把 `I`、`p` 下采样到 `1/s`（例如 s=4）；
2. 在小图上跑导向滤波，得到小的 `mean_a, mean_b`；
3. 把 `mean_a, mean_b` 上采样回原尺寸；
4. 用**原尺寸的 `I`** 做最后一步 `q = mean_a · I + mean_b`。

**加速 s² 倍**（比如 4×4 下采样加速 16 倍），效果几乎不损失。因为最后一步用的是**高清 I**，边缘细节仍然锐利。

## 8. 应用场景

### 8.1 保边平滑 / 磨皮

`I = p = 输入图`。适度的 r 和 ε 就能做出"皮肤光滑但眼睛毛发依然锐利"的效果。**几乎所有美颜 App 底层都在用它**。

### 8.2 抠图（Alpha Matting）

在 alpha 遮罩优化里，把粗糙的 alpha 当 `p`，原图当 `I`，一次导向滤波就能让 alpha 边缘对齐到彩色图的真实边界。这就是何恺明另一篇论文《Guided Image Filtering》的看家本领。

### 8.3 HDR 色调映射

**分层策略**：`I = p =` HDR 亮度图，用较大 r、较大 ε 得到**基础层（Base）**；`细节层 = I - Base`。压缩 Base 的动态范围，保留细节层，最后合成——**避开 halo 伪影**（这是双边滤波在此任务上的常见毛病）。

### 8.4 去雾

何恺明本人的另一名作**"暗通道先验去雾"**里，用导向滤波来精修透射率图 t(x)：粗略估计的 t 当 `p`，原始有雾图当 `I`——一步就能把 t 图的边缘对齐到景物边缘。

### 8.5 深度图上采样 / 联合上采样

RGBD 相机的深度图分辨率往往比彩色图低。**联合导向滤波**：`I =` 高清彩色图，`p =` 上采样后的深度图 → 输出 `q =` 边缘对齐到彩色图的高清深度图。

## 10. 常见疑问 FAQ

### Q1：为什么"局部线性"就一定保边？

因为线性变换 `q = aI + b` 的梯度 `∇q = a·∇I` 与 `∇I` **同方向、同位置**，只是被缩放。所以 `I` 里的边一定原封不动出现在 `q` 里。这是数学意义上的严格结论，不是启发式。

### Q2：ε 和窗口 r 该怎么选？

- **磨皮/去噪**：先固定 `r = 8`，试 `ε = 0.001, 0.01, 0.1`（[0,1] 图）看哪个最合适；
- **保结构上采样**：`r` 中等（4~8），`ε` 很小（如 1e-4），只让强边缘保住；
- **通用起点**：`r = 4~16`, `ε = 0.01² ~ 0.1²`。

### Q3：I 和 p 必须尺寸相同吗？

是的，导向滤波在**同尺寸**上做。如果 `p` 是低分辨率的（如深度图），需要先上采样到与 `I` 相同尺寸，再做导向滤波（或用**Fast Guided Filter** 的思路：小图算 a、b，上采样后用高清 I 合成）。

### Q4：为什么有"梯度反转"这个概念？双边滤波为啥有它？

双边滤波在**强边缘附近的中等梯度点**上，权重会突然被 range kernel 截断，导致输出的梯度方向**反过来**（本来递增的地方变递减）。视觉上表现为边缘**外侧的一条暗晕/亮晕**（halo）。**导向滤波因为是全局线性拟合，天然没有这个问题**——这是它比双边滤波更适合 HDR 色调映射的关键原因。

### Q5：导向滤波是不是就是加权最小二乘（WLS）的一种？

它的推导确实是最小二乘。但比 WLS 快得多（WLS 通常需要解稀疏线性系统），因为**导向滤波假设"每个窗口内线性"这个约束足够强**，直接给出闭式解，避免了全局求解。

### Q6：能不能推广到彩色引导图？

能。当 `I` 是 RGB 时，公式变成：

```
q_i = a_kᵀ · I_i + b_k
```

其中 `a_k` 是 3×1 向量、`I_i` 是 3×1 向量。求解需要一个 3×3 矩阵求逆（在每个窗口）。OpenCV 的 `guidedFilter` 已经支持彩色引导图，性能仍然是 O(N)。

### Q7：导向滤波会不会把噪声当成"结构"保下来？

会！如果 `ε` 太小、噪声很大，噪声在小窗口里可能造成较大方差，导致 `a → 1` 而**保住噪声**。**解决办法**：

1. 加大 `ε`（提高"边缘门槛"）；
2. 加大 `r`（大窗口里噪声被平均掉，方差主要来自真实边缘）；
3. 用**去噪版引导图**（比如先用 NLM 得到干净的 I，再用它引导滤波原图）。

### Q8：还有比导向滤波更强的保边滤波吗？

有：

- **加权最小二乘（WLS）**：更精细但慢；
- **域变换滤波（Domain Transform）**：Gastal 2011，也是 O(N) 保边；
- **深度学习去噪**（如 DnCNN、NAFNet）：效果碾压传统方法，但需要 GPU 和训练数据。

导向滤波胜在**简单、无需训练、可解释、极快**，仍然是很多产品的默认选项。

---

## 11. 一句话总结

> **导向滤波 = 一个假设（局部线性 `q = aI + b`）+ 一个代价函数（岭回归）+ 闭式解 `a = cov/(σ²+ε)` + 一个 box filter 平滑系数**。
>
> - 想要**保边平滑**？→ 令 `I = p`，选合适的 `r, ε`；
> - 想要**联合处理**（深度上采样/抠图/去雾）？→ 用彩色图当 `I`，粗糙目标当 `p`；
> - 想要**HDR 色调映射**？→ 用它做基础层-细节层分解，不会有 halo；
> - 想要**极快**？→ 全靠 box filter，加积分图 `O(N)`，加下采样再 `s²` 倍；
>
> 无论应用怎么变，核心都是那条 `q = aI + b` 和"方差决定保边强度"这两件事。**记住它们，你就掌握了导向滤波的灵魂**。
